# Where Can I Get Help or Report Issues?

When working with Magic and related components such as the Easy DingTalk SDK, having clear support avenues and knowing how to communicate problems effectively is crucial for timely resolution and continued productivity. This guide details your official support options, contact methods for security or enterprise concerns, community resources, and tips on how to prepare your requests for efficient assistance.

## Official Support Channels

To ensure that your issues, questions, or requests are addressed quickly and precisely, use the following official support avenues:

### 1. GitHub Issues

- For bugs, feature requests, and general development discussions related to Easy DingTalk SDK or Magic, open an issue at the GitHub repository:
  - Repository: [https://github.com/dtyq/magic](https://github.com/dtyq/magic)

- When submitting issues:
  - Clearly describe your problem or suggestion.
  - Include steps to reproduce the issue, if applicable.
  - Provide environment details (SDK version, PHP version, platform, etc.).
  - Share relevant error messages, logs, or stack traces.

### 2. GitHub Discussions

- For community questions, usage tips, and brainstorming, use the GitHub Discussions forum associated with the repository.
- This is the best place for guidance on usage patterns, integration scenarios, or sharing best practices.

  - Avoid public disclosure until acknowledged and resolved.

- **Enterprise or Business Inquiries:** For enterprise support, custom integration questions, or partnership discussions, contact `enterprise@magic.com`.

### 4. Official Documentation & Resources

- Always start by consulting the official docs and guides:

## Preparing Your Support Request

To get the fastest and most effective help, follow these best practices when requesting support:

- **Be Precise and Concise:** Clearly state your question or problem.

- **Include Environment Details:** SDK version, PHP version, operating system, and any relevant configurations.

- **Reproduction Steps:** Provide a minimal code snippet or steps to replicate your issue.

- **Error Messages:** Include full error messages, stack traces, or API response codes.

- **What Have You Tried:** Explain any troubleshooting steps or workarounds you've attempted.

- **Screenshots or Logs:** Attach relevant logs or screenshots when they help illustrate the problem.

By providing detailed information, you reduce back-and-forth communication and speed up resolution.
